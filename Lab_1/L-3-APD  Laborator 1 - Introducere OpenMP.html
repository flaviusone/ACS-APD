Laborator 1 - Introducere OpenMP

Laborator 1. Introducere in calculul paralel. OpenMP

OpenMP basics

  • este un API (Application Program Interface) pentru programarea paralelă
  • folosește o memorie partajata - shared memory
  • folosește threaduri dar maschează pornirea și oprirea lor (nu vedem funcții gen fork-join)
  • in C/C++ si Fortran
  • multiplatforma (Linux, Windows și OS X)
  • ofera o interfata simpla si flexibila pentru dezvoltarea aplicațiilor paralele.
  • contine directive, funcții și variabile de sistem
  • poate fi compilat cu gcc/intel Compilers/Visual Studio/clang
  • din cauza folosirii de directive #pragma codul va compila și executa serial dacă compilatorul nu suportă openmp

Directive OpenMP

#pragma omp directive-name [clause, ...] newline, unde:
    • #pragma omp este necesara pentru toate directivele OpenMP in C/C++
    • directive-name trebuie sa specifice un nume valid de directiva OpenMP.
    • [clause, ...] sunt optionale, pot aparea in orice ordine si repetate dupa necesitati.
    • newline este absolut necesara.
    • Dupa linia noua urmeaza blocul structurat ce este referit de aceasta directiva.

  • Se urmează convențiile și standardele compilatoarelor de C/C++
  • Sunt case sensitive
  • Directivele "lungi" pot fi continuate pe liniile succesive prin adaugarea unui \ la sfarsitul primei linii a directivei.

directive-name:

    • parallel: un bloc ce va fi executat in paralel de OMP_NUM_THREADS thread-uri. Este constructia fundamentala in OpenMP.
  • Structuri Work-sharing:
    • for - partajeaza o iteratie a unui ciclu peste toate thred-urile (data parallelism);
    • sections - sparge codul in sectiuni separate (functional parallelism);
      • secțiunile sunt identificate cu #pragma omp section
    • single - serializează o portiune de cod.
  • Constructii de sincronizare:
    • master- doar thread-ul master va executa aceasta regiune de cod;
    • critical- regiunea respectiva de cod va fi executata doar de un singur thread;
    • barrier- toate threadurile din pool se sincronizeaza;
    • atomic- o anume regiune de memorie se va updata in mod atomic - un fel de mini sectiune critica;

clause:

    • private(lista variabile) - variabilele din lista sunt private in fiecare thread.
    • shared(lista variabile)- variabilele din lista sunt partajate intre thread-urile team-ului curent;
    • default(private|shared|none) - permite utilizatorului sa seteze un default "PRIVATE", "SHARED", sau "NONE" pentru toate variabilele dintr-o regiune paralela;
    • firstprivate(lista variabile)- combina functionalitatea clauzei PRIVATE cu initializarea automatizata a variabilelor din lista: initializarea variabilelor locale se face cu valoarea anterioara intrarii in ciclu;
    • lastprivate(lista variabile) - combina functionalitatea clauzei PRIVATE cu o copie a ultimei iteratii din sectiunea curenta;
    • reduction(operatie:lista variabile)- realizeaza o reducere pe variabilele care apar in lista (cu o operatie: + - * /, etc).
    • num_threads(number)
    • schedule(type:[chunk_size]) - type poate fi dynamic(se dau iteratii la threaduri in functie de cum termina), static (stabilit de la inceput) sau guided(se dau iteratii la threaduri in functie de cum termina, micsorand in timp marimea chunksizeului)
    • if(scalar_expression) - porneste threadurile doar daca rezultatul e true

Ca o simplificare se poate folosi o directivă de genul

#pragma omp parallel for

#pragma omp parallel sections

Cum se compilează și rulează OpenMP?

In decursul laboratoarelor si temelor vom lucra cu OpenMP in 2 forme:

  • local - pe calculatoarele din laborator sau cele personale. Pentru a compila local surse C/C++ cu suport OpenMP:
    • gcc -fopenmp my_program.c -o my_program
  • pe cluster - care poate fi accesat prin fep.grid.pub.ro. - poate necesita recompilare pe cluster
    • Pentru a ajunge pe fep, utilizati
      • ssh -l username fep.grid.pub.ro -v
    • Pentru a copia fisiere pe fep, utilizati
      • scp file_name username@fep.grid.pub.ro:.

    • Pentru a rula pe cluster

      • qsub -cwd -q queue_name script_name

    • Pentru a verifica execuția
      • qstat

Pentru a seta numărul de thread-uri folosiți funcția omp_set_num_threads(int nr_threads):

Similar, se poate seta in linie de comanda variabila de mediu OMP_NUM_THREADS:
  • export OMP_NUM_THREADS=4

Reminder debugging Linux

  • Compilarea se face adăugând -g în linia de comandă gcc

  • gdb executable_name pentru pornirea debugger-ului
    • file executable_name identificare executabilului din interiorul gdb
    • break functioname setarea punctului de break la începutul funcției functioname
    • break filename:linenumber setarea punctului de break la linia linenumber din fișierul filename
    • info threads afișează informații despre thread-uri
    • print expression afișează rezultatul expresiei, poate fi o variabilă

Întrebări importante

  1. Câte procesoare/core-uri am la dispoziție?
    • cat /proc/cpuinfo
  2. Ce versiune de OpenMP am la dispoziție?
    • gcc -v + comparatie site web gcc
  3. Cum măsor scalabilitatea?
    • time nume_executabil multiple măsurători cu număr diferit de thread-uri
  4. Am pornit cu adevărat mai multe threaduri?
    • gdb se pornește aplicația se setează un break point se verifică threadu-urile
  5. Chiar dacă am mai multe threaduri pornite, folosesc numărul de core-uri dorit de mine?
    • htop
